A code trace displays the function name and line number to the Command Window whenever a specific line of code is executed. It can also display the value of a value or expression within the executing function.
The following examples illustrate the use of code traces using the function fib, which is a recursive implementation of a Fibonacci number generator.
function out = fib(n)
if n == 0
out = 0;
elseif n == 1
out = 1;
else
out = fib(n-1) + fib(n-2);
end

Display function name and line number

This example shows how to display the function name and the line number whenever a specific code line is executed. First, add the code trace.
addCodeTrace("fib",2)
Next, call the function fib:
fib(3)
[trace] fib:2 [trace] fib:2 [trace] fib:2 [trace] fib:2 [trace] fib:2
ans = 2
The text "[trace]" distinguishes code trace output from other output that a function might display to the Command Window. On each trace line, the function name and line number are indented based on the depth of the function call stack at the time the line was executed. This helps to visualize the recursive nature of the fib implementation.

List code traces currently in effect

This example shows how to list the code traces currently in effect. First, let's add a second code trace.
addCodeTrace("fib",5)
Call fib to see the effect of the second code trace.
fib(3)
[trace] fib:2 [trace] fib:2 [trace] fib:2 [trace] fib:5 [trace] fib:2 [trace] fib:2 [trace] fib:5
ans = 2
Now, use codeTraces to get a table that shows the current code traces.
traces = codeTraces
traces = 2×2 table
 FunctionNameLineNumber
1"fib"2
2"fib"5

Clear code traces

This example shows how to clear all code traces in effect using the function clearCodeTraces.
clearCodeTraces
If you call codeTraces after clearing all code traces, you get an empty table.
codeTraces
ans = 0×2 empty table

Display the value of a variable or expression

This example shows how to display the value of a variable or expression in the code trace output. You can do this by passing in the variable name or expression to addCodeTrace as an Expression parameter.
addCodeTrace("fib",2,Expression = "n")
The expression "n" will be evaluated before line 2 is executed, and it is evaluated using the context and workspace of the function fib.
fib(3)
[trace] fib:2 n = 3 [trace] fib:2 n = 2 [trace] fib:2 n = 1 [trace] fib:2 n = 0 [trace] fib:2 n = 1
ans = 2

Display a label

This example shows how to display a user-specified text label in the code trace output. You can do this by passing in the label to addCodeTrace as a Label parameter.
Used in combination with the ability to display a variable or expression, we can get a comprehensive view of how the recursive implementation of fib works.
clearCodeTraces
addCodeTrace("fib",2,Label = "function entry", Expression = "n")
addCodeTrace("fib",7,Label = "recursive function call")
addCodeTrace("fib",3,Label = "function exit, returning 0")
addCodeTrace("fib",5,Label = "function exit, returning 1")
 
fib(4)
[trace] fib:2 function entry n = 4 [trace] fib:7 recursive function call [trace] fib:2 function entry n = 3 [trace] fib:7 recursive function call [trace] fib:2 function entry n = 2 [trace] fib:7 recursive function call [trace] fib:2 function entry n = 1 [trace] fib:5 function exit, returning 1 [trace] fib:2 function entry n = 0 [trace] fib:3 function exit, returning 0 [trace] fib:2 function entry n = 1 [trace] fib:5 function exit, returning 1 [trace] fib:2 function entry n = 2 [trace] fib:7 recursive function call [trace] fib:2 function entry n = 1 [trace] fib:5 function exit, returning 1 [trace] fib:2 function entry n = 0 [trace] fib:3 function exit, returning 0
ans = 3
Clean up.
clearCodeTraces
